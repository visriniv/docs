When a question matches a known shape, ORCA chat automatically reaches for a pre-built **skill** — a consultant's playbook with pre-validated Oracle calls — before improvising. Three ship today:

<CardGroup cols={1}>
  <Card title="Fusion Security Posture" icon="user-shield">
    Role assignments, data access, security profiles, and AOR for a user.
  </Card>
  <Card title="Worker Scheduling" icon="calendar-days">
    What shift someone works, schedules, rosters, and who's on a given day.
  </Card>
  <Card title="Workforce Counts" icon="users">
    Headcount and worker-type breakdowns.
  </Card>
</CardGroup>

## How skills are cited

The answer cites the calls it made **inline in its own text** — the skills require it to quote the exact REST call or query as evidence (even "no rows" is stated as evidence). There is no separate citations panel, so read the prose for the sourcing.

<Note>
  Skills only appear once the connection's REST catalog has been discovered. If discovery hasn't run, the chat won't have skills to reach for.
</Note>
