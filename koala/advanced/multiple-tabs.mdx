## Overview

Koala Data Explorer supports multiple query tabs, allowing you to work on several SQL queries simultaneously. This feature improves productivity by letting you switch between different queries without losing your work.

<Note>
  Multiple tabs are available in the paid version of Koala Data Explorer. The free version supports only one active tab.
</Note>

## Working with Tabs

### Creating New Tabs

Create additional query tabs:

1. **Keyboard shortcut**: Press `Ctrl+T` (Windows/Linux) or `Cmd+T` (Mac)
2. **Menu option**: Use "New Tab" from the Koala menu
3. **Button**: Click the "+" button in the tab bar
4. **Context menu**: Right-click in tab area and select "New Tab"

Each new tab provides a fresh query editor with its own:
- SQL editor space
- Execution context
- Results area
- Query history

### Tab Navigation

Switch between tabs easily:

- **Click tabs**: Click on any tab to switch to it
- **Keyboard shortcuts**: 
  - `Ctrl+Tab`: Next tab
  - `Ctrl+Shift+Tab`: Previous tab
  - `Ctrl+1-9`: Jump to specific tab by number
- **Mouse wheel**: Scroll over tab bar to navigate

### Tab Management

Manage your open tabs:

- **Close tabs**: Click the "Ã—" on any tab or press `Ctrl+W`
- **Reorder tabs**: Drag tabs to rearrange them
- **Pin tabs**: Keep important tabs open (planned feature)
- **Tab titles**: Tabs show query content preview or custom names

## Tab Features

### Independent Contexts

Each tab maintains its own:

- **Query content**: Separate SQL editor
- **Execution state**: Independent query running
- **Results display**: Separate results panel
- **Undo/redo history**: Per-tab editing history
- **Cursor position**: Tab-specific editor state

### Tab Titles

Tab titles automatically update based on:
- **Query content**: First line of SQL or table names
- **File names**: If query is saved to file
- **Custom names**: User-defined tab names (planned)
- **Status indicators**: Modified, running, or error states

### Tab States

Visual indicators show tab status:
- **Clean tabs**: No unsaved changes
- **Modified tabs**: Asterisk (*) for unsaved changes
- **Running tabs**: Spinner when query is executing
- **Error tabs**: Warning icon for queries with errors

## Productivity Features

### Parallel Development

Multiple tabs enable:

1. **Compare queries**: Different approaches side-by-side
2. **Reference data**: Keep lookup queries open
3. **Iterative development**: Test variations of same query
4. **Multi-table exploration**: Explore different tables simultaneously

### Workflow Organization

Organize work across tabs:

- **Main development**: Primary query in main tab
- **Reference queries**: Supporting data in other tabs
- **Testing**: Experimental queries in separate tabs
- **Documentation**: Query examples and templates

### Tab-Specific Results

Each tab has independent results:
- **Separate results panels**: Each tab shows its own query results
- **Independent exports**: Export results from any tab
- **Result history**: Results preserved when switching tabs
- **Sorting and filtering**: Per-tab result manipulation

## Tab Limitations

### Performance Considerations

With multiple tabs:
- **Memory usage**: Each tab uses memory for editor and results
- **Connection sharing**: All tabs share same database connection
- **Resource limits**: Too many tabs may impact performance
- **Result caching**: Large results in multiple tabs use more memory

### Recommended Limits

For optimal performance:
- **5-10 tabs**: Recommended maximum for most use cases
- **Close unused tabs**: Regular cleanup of old tabs
- **Clear large results**: Remove large result sets when done
- **Monitor memory**: Watch VS Code memory usage

## Tab Keyboard Shortcuts

### Navigation Shortcuts

| Action | Windows/Linux | Mac | Description |
|--------|--------------|-----|-------------|
| New Tab | `Ctrl+T` | `Cmd+T` | Create new query tab |
| Close Tab | `Ctrl+W` | `Cmd+W` | Close current tab |
| Next Tab | `Ctrl+Tab` | `Cmd+Tab` | Switch to next tab |
| Previous Tab | `Ctrl+Shift+Tab` | `Cmd+Shift+Tab` | Switch to previous tab |

| Action | Windows/Linux | Mac | Description |
|--------|--------------|-----|-------------|

| Close All | `Ctrl+Shift+W` | `Cmd+Shift+W` | Close all tabs |

## Best Practices

### Tab Organization

Organize tabs effectively:

1. **Logical grouping**: Related queries in adjacent tabs
2. **Clear naming**: Use descriptive SQL content
3. **Regular cleanup**: Close completed or unnecessary tabs
4. **Primary tab**: Keep main work in first tab

### Development Workflow

Effective multi-tab workflow:

1. **Main query**: Primary development in tab 1
2. **Reference data**: Supporting queries in tabs 2-3
3. **Testing**: Experimental variations in additional tabs
4. **Documentation**: Examples and templates in final tabs

### Performance Management

## Tab Settings

### Configuration Options

Configure tab behavior through VS Code settings:

```json
{
  "koalaDataExplorer.tabs.maxTabs": 10,
  "koalaDataExplorer.tabs.autoClose": false,
  "koalaDataExplorer.tabs.showModified": true,
  "koalaDataExplorer.tabs.confirmClose": true
}
```

### Tab Preferences

Customize tab experience:
- **Maximum tabs**: Limit number of open tabs
- **Auto-close**: Automatically close unused tabs
- **Modified indicators**: Show unsaved changes
- **Close confirmation**: Confirm before closing modified tabs

## Troubleshooting

### Tab Issues

Common tab problems:

#### "Too many tabs open"
- Close unused tabs
- Save important queries to files
- Restart VS Code if needed

#### "Tab not responding"
- Check if query is still running
- Cancel long-running queries
- Close and recreate problematic tab

#### "Performance slow with tabs"
- Reduce number of open tabs
- Clear large result sets
- Check VS Code memory usage

### Memory Management

Optimize tab memory usage:

1. **Regular cleanup**: Close old tabs
2. **Result management**: Clear large results
3. **Query optimization**: Use row limits
4. **Restart when needed**: Restart VS Code periodically
